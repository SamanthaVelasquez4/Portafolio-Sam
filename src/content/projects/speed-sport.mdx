Speed Sport es una Webde delivery para ropa deportiva, funciona de manera similar a Yuuju! Donde tiendas como Nike y Adidas 
podrían contratar los servicios de Speed Sport para hacer los pedidos de los clientes y llevar dichos pedidos a sus casas.  

## Consta de cuatro partes: 
>Frontend para clientes: https://github.com/2TIN-8B/Frontend-appClientes  
>Frontend para conductores: https://github.com/SamanthaVelasquez4/Frontend-app-motoristas-repartidores  
>Frontend para administradores: https://github.com/SamanthaVelasquez4/Frontend-web-administrativa  
>Backend: https://github.com/SamanthaVelasquez4/Backend-SpeedSport 

## Esta desarrollado en: 
<li> Node.js + TypeScript para el backend </li>
<li> HTML, CSS, Bootstrap y JavaScript para el frontend </li>
<li> MongoDB para la base de datos </li>

## Experiencia adquirida: 

Adquirí experiencia integral en desarrollo full-stack, trabajando con múltiples frontends (clientes, motoristas y administradores) y 
un backend unificado. Mi participación incluyó: 

### 1. Arquitectura Modular y Escalable

<li> Diseñé tres frontends independientes para cada rol (cliente, motorista, administrador), optimizando la experiencia de usuario según 
sus necesidades. </li>

<li> Implementé un backend en Node.js + TypeScript con arquitectura RESTful, asegurando conexiones eficientes con la base de datos (MongoDB) 
y los frontends. </li>

### 2. Gestión de Roles y Flujos de Trabajo

<li> Clientes: Interfaz intuitiva para comprar, rastrear pedidos y gestionar devoluciones. </li>

<li> Motoristas: Elección de pedidos y actualización de estados de entrega. </li>

<li> Administradores: Gestión de tiendas y resolución de incidencias. </li>

### 3. Buenas Prácticas

<li> Apliqué TypeScript en el backend para tipado estático y reducir errores. </li>

<li> Versionamiento con Git/GitHub, manejando múltiples repositorios de forma coordinada. </li>

### 4. Bases de datos no relacionales

<li> Reforcé mis habilidades en el manejo de bases de datos NoSQL con MongoDB, junto con la librería Mongoose para la conexión y 
gestión de datos en el backend. </li>
