Lideré el diseño e implementación de un modelo de base de datos relacional en Oracle inspirado en la arquitectura de Amazon, 
abarcando módulos clave como gestión de usuarios, productos, métodos de pago y transacciones de compra. Aunque el proyecto 
no cubrió la totalidad de los servicios de Amazon, se priorizó la creación de una estructura escalable que integrara: 

<li> Usuarios: Registro, autenticación y perfiles (compradores/vendedores). </li>

<li>Productos: Catálogo con categorías, inventario y reseñas.</li> 

<li>Proceso de compra: Carrito, órdenes, historial y estados de envío. </li>

<li>También implementamos un backend en Node.js para exponer APIs RESTful que interactuaran con la base de datos. </li>

## Experiencia adquirida: 

### Trabajo con Bases de Datos Relacionales (Oracle): 

<li>Diseño de esquemas complejos con relaciones (1:N, N:M, 1:1). </li>

<li>Uso de PL/SQL para triggers y procedimientos almacenados. </li>

### Desarrollo de APIs con Node.js: 

<li>Creación de endpoints RESTful con Express. </li>

<li>Manejo de conexiones asíncronas a Oracle (oracledb). </li>

<li>Este proyecto combinó mis habilidades en modelado de bases de datos (Oracle) y desarrollo backend (Node.js) p
    ara replicar la lógica central de un e-commerce como Amazon. Aprendí a diseñar esquemas relacionales eficientes y 
    a construir APIs seguras y optimizadas, trabajando en equipo para garantizar la integración fluida entre capas. </li>
